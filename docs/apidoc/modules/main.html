Module main

This is the entry point of the widget API. Exported functions of this module are available in the global useeio variable when the widget library is imported. It re-exports some other modules so that these are then also available under the useeio variable.

Type aliases

DemandType

DemandType: "Consumption" | "Production"

Describes the type of a demand vector. This is equivalent to the analysis type in the SMM tools. A demand vector of the type "Consumption" includes the final demand of households, government, etc. wheras a demand vector of the type "Production" focuses on the production of goods and services.

ResultPerspective

ResultPerspective: "direct" | "intermediate" | "final"

Describes the perspective of a calculation result:

  • direct: direct contribution results: D * diag(s)
  • intermediate: direct + upstream results at each point of the supply chain (without loop correction): D * L * diag(s)
  • final: final results related to the demand vector: D * L * diag(d)

model

  • model(conf: WebApiConfig): Model

  • Creates a new model for the given web-API configuration. A Model instance caches the results of API requests and provides additional functions like aggregating multi-regional sectors. Different widgets that access the same web-API should use the same Model instance for efficiency reasons. Creating a model instance in JavaScript looks like this:

    var model = useeio.model({
    endpoint: './api',
    model: modelID,
    asJsonFiles: true,
});

    Parameters

    • conf: WebApiConfig

    Returns Model
